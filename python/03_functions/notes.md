# Пользовательские функции - Конспект

## Конспект по теме: "Функции без параметров"

### Основные понятия:

Ранее я использовал встроенные в **Python** функции `print()`, `input()`, `int()`, `str()`, `len()` и множество других, но пришло время начать писать свои собственные функции.

Начнём с того, что же такое функция в языке **Python**.

**Функция** - `def` в **Python** - это фрагмент кода, который выполняет определённые операции и возвращает результат этих операций. Она является отдельной, функционально независимой частью программы, выполняющей определённую задачу.

Вся суть в том, что его можно написать один раз и переиспользовать во всей программе.

Приведу простой пример..

Допустим необходимо решить задачу в которой требуется изобразить звёздный прямоугольник размерами 5 строк на 7 столбцов.

- Так бы выглядел самый очевидный вариант кода без использования функций:
  ```python
  print('*******')
  print('*******')
  print('*******')
  print('*******')
  print('*******')
  ```
- Далее, если вспомнить операторы умножения строк на числа, можно было бы придти к такому коду:
  ```python
  print('*' * 7)
  print('*' * 7)
  print('*' * 7)
  print('*' * 7)
  print('*' * 7)
  ```
- И наконец, если вспомнить и про циклы, то код принимает вид:
  ```python
  for _ in range(5):
      print('*' * 7)
  ```

А теперь представим, просто представим, что нам требуется изобразить сразу несколько таких фигур, например 4 штуки.

- Тогда нам пришлось бы повторять данный цикл 4 раза:
  ```python
  for _ in range(5):
      print('*' * 7)

  print()

  for _ in range(5):
      print('*' * 7)

  print()

  for _ in range(5):
      print('*' * 7)

  print()

  for _ in range(5):
      print('*' * 7)
  ```

  Результатом выполнения такого кода будет:
  ```python
  *******
  *******
  *******
  *******
  *******
  
  *******
  *******
  *******
  *******
  *******
  
  *******
  *******
  *******
  *******
  *******

  *******
  *******
  *******
  *******
  *******
  ```
Вроде как данный код вполне себе решает поставленную задачу, но у него есть несколько недостатков. 

Во-первых, данный код может показаться довольно громоздким из-за нескольких повторений одной и той же части кода, отвечающей за вывод прямоугольника.

Во-вторых, если понадобится изменить размеры прямоугольника, придётся менять их аж 4 раза, в каждой части кода, отвечающей за вывод прямоугольника.

Вот здесь нам и пригодятся функции и вместо повторения кода для вывода прямоугольника, можно перенести его в **отдельную функцию** и вызвать её 4 раза.

- Для создания функции напишем такой код:
  ```python
  def draw_box():
      for _ in range(5):
          print('*' * 7)
  ```

- Когда функция создана, чтобы увидеть результат её работы, надо вызвать её по имени:
  ```python
  draw_box()
  ```

- Теперь, чтобы изобразить 4 прямоугольника, можно написать код:
  ```python
  draw_box()
  print()
  draw_box()
  print()
  draw_box()
  print()
  draw_box()
  ```

Благодаря использованию функции код стал короче, читабельнее, а главное, если потребуются иные размеры прямоугольника, достаточно будет изменить только саму функцию `draw_box()`.



#### Именование функций 

Имена функциям назначаются точно также, как переменным. Имя функции должно быть достаточно описательным, чтобы любой человек, который читает или будет читать ваш код, мог догадаться, что именно делает та или иная функция.

**Python** и тут требует соблюдения тех же правил, что при именовании переменных:
1. В имени функции используются **только латинские буквы** a-z, A-Z, цифры и символ нижнего подчеркивания (_);
2. Имя функции **не может** начинаться с цифры;
3. Имя функции **по возможности должно отражать её назначение**;
4. Символы **верхнего** и **нижнего** регистра **различаются**.

Следует помнить, что **Python** это регистрочувствительный язык. Для именования переменных и функций принято использовать стиль **lower_case_with_underscores** (слова из маленьких букв с подчеркиваниями).

Поскольку функции **выполняют действия**, большая часть программистов предпочитает в их именовании использовать **глаголы**.

Например:
- функцию, которая рисует прямоугольник, можно назвать `draw_box()`;
- функцию, которая печатает чек, можно назвать `print_check()`;
- функцию, которая вычисляет заработную плату до удержаний, можно назвать `calculate_gross_pay()`.

Каждое из этих имён даёт описание того, что делает функция.#### Именование функций

#### Синтаксис и объявление функций

В **Python**, как и в других языках программирования, есть особые правила для создания функций. Если их не соблюдать, то интерпретатор не сможет правильно обработать код и, скорее всего, выдаст ошибку.

Для объявления функции используют **ключевое слово `def`**(от англ. define - определить, обозначить).

- В общем виде объявление функции выглядит следующим образом:
  ```python
  def имя_функции (аргументы):
      тело_функции
      return результат
  ```
Здесь:
- `имя_функции` - название, с помощью которого можно вызывать функцию в коде;
- `аргументы` - значения, которые функция принимает на вход. Это поле может быть пустым;
- `тело_функции` - набор инструкций, которые выполняются при вызове;
- `результат` - значения, которые функция возвращает при завершении работы.


- Рассмотрим на примере функцию, которая находит сумму двух чисел и возвращает результат вычислений:
  ```python
  def sum(a, b):
      return a + b
  ```

Также при объявлении функции очень важно убедиться в том, что каждая строка тела функции начинается с **одинакового количества пробелов**, иначе произойдёт ошибка.

- Приведённый ниже код:
  ```python
    def print_greeting():
        print('Доброе утро!')
  print('Сегодня мы будем изучать функции.')
        print('Это мегаважная тема')
  ```
  приводит к возникновению ошибки:
  ```python
  IndentationError: unexpected indent
  ```

  

#### Вызов функции

Как вызвать функцию?

Чтобы вызвать функцию, надо ввести её название и передать аргументы в скобках.

- В общем виде синтаксис вызова выглядит вот так:
  ```python
  имя_функции(аргументы)
  ```

Теперь вызовем функцию для нахождения суммы, которая была создана ранее.

- Вот как это будет выглядеть:
  ```python
  c = sum(a, b)
  ```

Результат значения функции будет записан в переменную `c`.

Стоит помнить, что объявление функции должно **предшествовать** её вызову, т.е. при вызове функции необходимо убедиться, что эта функция была объявлена до вызова.

- Приведённый ниже код:
  ```python
  # вызов
  print_message()

  # объявление
  def print_message():
      print('Я - Андрей,')
      print('санстрайк. ')
  ```
  приведёт к возникновению ошибки:
  ```python
  NameError: name 'print_message' is not defined
  ```
  

## Конспект по теме: "Функции с параметрами

### Основные понятия:

- Вспомним функцию `draw_box()`, которая выводит звёздный прямоугольник с размерами 5 на 7:
  ```python
  def draw_box():
      for i in range(5):
          print('*' * 7)
  ```

Было бы гораздо удобнее, если бы эта функция выводила прямоугольник с произвольными размерами. 

И функции как раз таки могут принимать входные параметры, что делает их более гибкими.

- **Функции с параметрами** объявляются **так же** как и функции без параметров, только с указанием в скобках:
  ```python
  def название_функции(параметры):
      блок кода
  ```

Теперь, зная это, можно переписать предыдущую версию функции `draw_box()` так, чтобы она принимала параметры, задающие высоту и ширину прямоугольника:

- Вот как это будет выглядеть:
  ```python
  def draw_box(height, width): # функция принимает 2 параметра
      for i in range(height):
          print('*' * width)
  ```

Теперь функция `draw_box()` принимает два целочисленных параметра `height` - высота прямоугольника и `width` - ширина прямоугольника, и для её вызова нам нужно обязательно их указать.

- И теперь, чтобы вывести звездный прямоугольник размерами 10 на 15 пишем код:
  ```python
  draw_box(10, 15)
  ```
  выводит:
  ```python
  ***************
  ***************
  ***************
  ***************
  ***************
  ***************
  ***************
  ***************
  ***************
  ***************
  ```

Теперь с помощью данной функции можем в одной программе выводить прямоугольники разных размеров.

- Приведённый ниже код:
  ```python
  draw_box(3, 3)
  print()
  draw_box(5, 5)
  print()
  draw_box(4, 10)
  ```
  выводит:
  ```python
  ***
  ***
  ***

  *****
  *****
  *****
  *****
  *****

  **********
  **********
  **********
  **********
  ```

На место параметров можно подставлять не только целочисленные константы, но и значения переменных:

- Приведённый ниже код:
  ```python
  n = 3
  m = 9
  draw_box(n, m)
  ```
  выводит:
  ```python
  *********
  *********
  *********
  ```

Приведу ещё один пример:

- Функцию `print_hello(n)`, которая принимает одно натуральное число `n` и печатает слово **hello** ровно `n` раз:
  ```python
  def print_hello(n):
      print('Hello' * n)
  ```

- Приведённый ниже код:
  ```python
  print_hello(3)
  print_hello(5)
  times = 2
  print_hello(times)
  ```
  выводит:
  ```python
  HelloHelloHello
  HelloHelloHelloHelloHello
  HelloHello
  ```

Эту функцию можно сделать ещё более гибкой, если передавать в неё ещё один параметр - текст для вывода.

- Вот как это будет выглядеть:
  ```python
  def print_text(text, n):
      print(text * n)
  ```

- Приведённый ниже код:
  ```python
  print_text('Hello', 5)
  print_text('A', 10)
  ```
  выводит:
  ```python
  HelloHelloHelloHelloHello
  AAAAAAAAAA
  ```

Параметры функций часто называют **параметрическими переменными**.

### Разница между аргументами и параметрами

**Аргумент** - это **любая порция данных**, которая передаётся в функцию, когда функция **вызывается**. 

**Параметр** - это **переменая**, которая **получает аргумент**, переданный в функцию.

- Для функции `draw_box(height, width)`:
  ```python
  def draw_box(height, width):
      for i in range(height):
          print('*' * width)
  ```

Параметрами являются переменные `height` и `width`.

- И в момент вызова данной функции:
  ```python
  height = 10
  draw_box(height, 9)
  ```
  аргументами являются `height` и `9`.

Иногда вместо параметров и аргументов говорят о **формальных параметрах** и **фактических параметрах**. 

**Формальные параметры** - это переменные, которые пишутся при описании функции.

**Фактические параметры** - это то, что реально подставляется при вызове функции.

### Внесение изменений в параметры

Когда **аргумент** передаётся в функцию, параметрическая переменная функции будет ссылаться на значение этого аргумента. Однако любые изменения, которые вносятся в параметрическую переменную, не будут влиять на аргумент.

- Приведённый ниже код:
  ```python
  def draw_box(height, width):
      height = 2
      width = 10
      for i in range(height):
          print('*' * width)
  n = 5
  m = 7
  draw_box(n, m)
  print(n, m)
  ```
  выводит:
  ```python
  **********
  **********
  5 7
  ```

В теле функции вносятся изменения в значения параметрических переменных `height` и `width`, однако это никак не повлияло на значение переменных `n` и `m` из основной программы, которые передавались в качестве аргументов в функцию `draw_box()`.

Т.е. если в теле функции уже заранее заданы значения параметрических переменных, то даже если передать новые значения в функцию, функция всё-равно перезапишет те значения, что были внутри неё и использует именно их, а новые потеряются.

## Конспект по теме: "Глобальные и локальные переменные"

### Основные понятия:

### Локальные переменные

**Локальными** называются переменные, **объявленные внутри функции** и доступные **только ей самой**.

Программный код, который находится за пределами функции **не имеет доступа к локальным переменным**.

- Рассмотрим функцию `print_ekaterinburg()`, которая выводит информацию о количестве птиц, обитающих в Екатеринбурге:
  ```python
  def print_ekaterinburg():
      birds_count = 3400
      print('В Екатеринбурге обитает', birds_count, 'птиц.')
  ```

Внутри тела функции была создана переменная `birds_count`, которой присваивается значение `3400`. Такая переменная **является локальной для функции `print_ekaterinburg()`**.

Каждый раз, когда переменной внутри функции присваивается значение, в результате **создаётся локальная переменная**.

Она принадлежит функции, в которой была создана и к ней получает доступ только программный код этой функции.

Т.е. она может использоваться только внутри той функции, где была создана.

Если программный код одной функции попытается обратится к **локальной переменной**, принадлежащей другой функции, произойдёт ошибка.

- Рассмотрим пример:
  ```python
  def print_ekaterinburg():
      birds_count = 3400
      print('В Екатеринбурге обитает', birds_count, 'птиц.')

  def print_moscow():
      print('В Москве обитает', birds_count, 'птиц.')
  ```
  выводит:
  ```python
  NameError: name 'birds_count' is not defined
  ```

Функция `print_moscow()` обращается к локальной переменной `birds_count`, которая является локальной переменной внутри функции `print_ekaterinburg()`. 

И так как внутри функции `print_moscow()` не определена данная переменная, то вызов приводит к ошибке.

**Локальные переменные скрыты от других функций**, поэтому другие функции **могут иметь собственные локальные переменные с тем же именем**. 

- Например:
  ```python
  def print_ekaterinburg():
      birds_count = 3400
      print('В Екатеринбурге обитает', birds_count, 'птиц.')

  def print_moscow():
      birds_count = 4000
      print('В Москве обитает', birds_count, 'птиц.')
  ```

В каждой из этих двух функций есть локальная переменная с одним и тем же именем - `birds_count`. 

Но они **никогда не видны одновременно**, так как находятся в разных функциях.

Т.е. когда у нас выполняется функция `print_ekaterinburg()`, видна переменная `birds_count`, значение которой равно `3400`.

А когда выполняется функция `pring_moscow()`, видна переменная `birds_count`, значение которой равно `4000`.

Получается, что **разные функции** могут иметь **локальные переменные с одинаковыми именами**, потому что они не имеют доступа к локальным переменным друг-друга.

### Область видимости переменной

**Область видимости переменной** - это часть программы, в которой к ней можно обращаться(та функция, где была создана эта переменная).

Переменная видима программному коду только в области её видимости.

**Никакая инструкция за пределами функции не может обращаться к такой переменной**.

К локальной переменной также не может обращаться программный код, который появляется внутри функции **до того**, как переменная была создана.

- Например, если в функции `pring_ekaterinburg()` поменять местами строки кода:
  ```python
  def print_ekaterinburg():
      print('В Екатеринбурге обитает', birds_count, 'птиц.')
      birds_count = 3400
  ```
  то при вызове данной функции получим ошибку:
  ```python
  UnboundLocalError: local variable 'birds_count' referenced before assignment
  ```

Ошибка возникла потому, что при попытке обратиться к локальной переменной - код не нашёл её, так как она была объявлена только после обращения.

#### Область видимости параметрической переменной

Областью видимости параметрической переменной является функция, которая использует данную параметрическую переменную.

К параметрической переменной имеет доступ весь программный код этой функции.

- Рассмотрим такую функцию:
  ```python
  def draw_box(height, width):
      for i in range(height):
          print('*' * width)
  ```

Параметрические переменные тут `height` и `width`. Внутри функции объявляется одна локальная переменная `i`.

Параметрическая переменная также является **локальной**.

### Глобальные переменные

**Глобальными** называются переменные, объявленные в **основной программе** и доступные как программе, так и **всем её функциям**.

- Рассмотрим такой код:
  ```python
  average_salary = 100000

  def print_moscow():
      print('Средняя зарплата в Москве:', average_salary, 'рублей.')

  def print_ekaterinburg():
      print('Средняя зарплата в Екатеринбурге:', average_salary, 'рублей.')
  ```

В самом начале данной программы была создана глобальная переменная - `average_salary`, значение которой равно `100000`. 
Далее были созданы 2 функции, которые обращаются к **глобальной переменной**.

- Приведённый ниже код:
  ```python
  print_moscow()
  print_ekaterinburg()
  ```
  выводит:
  ```python
  Средняя зарплата в Москве: 100000 рублей.
  Средняя зарплата в Екатеринбурге: 100000 рублей.
  ```

Обмен информацией между основной программой и функциями осуществляется **только с помощью параметров функций и глобальных переменных**.

Функция может использовать **любые глобальные переменные кроме имеющих те же имена, что и её локальные переменные**.

Если в функции объявлена локальная переменная с тем же именем, что у одной из глобальных, то данная глобальная переменная становится **недоступной в этой функции**, и при указании идентификатора переменной произойдёт обращение к **локальной переменной функции**, а не одноимённой глобальной.

- Рассмотрим такой пример:
  ```python
  average_salary = 100000

  def print_moscow():
      average_salary = 200000
      print('Средняя зарплата в Москве:', average_salary, 'рублей.')
  
  def print_ekaterinburg():
      average_salary = 80000
      print('Средняя зарплата в Екатеринбурге:', average_salary, 'рублей.')
  ```
  при вызове функций:
  ```python
  print_moscow()
  print_ekaterinburg()
  ```
  вывод будет такой:
  ```python
  Средняя зарплата в Москве: 200000 рублей.
  Средняя зарплата в Екатеринбурге: 80000 рублей.
  ```
Почему так получилось?

В самом начале программы была создана глобальная переменная `average_salary`, значение которой равно `100000`. Далее были созданы две функции, в которых были объявлены локальные переменные с **таким же именем `average_salary`**.

И таким образом при обращении к переменной `average_salary` внутри функций будет происходить обращение **именно к локальной переменной**.

#### Ключевое слово `global`

Если нужно, чтобы инструкция внутри функции присваивала значение **глобальной переменной**, то требуется дополнительный шаг.

В этом случае **глобальная переменная** должна быть объявлена **внутри функции**.

- Пример:
  ```python
  def print_moscow():
      global average_salary
      average_salary = 150000
      print('Средняя зарплата в Москве:', average_salary, 'рублей.')

  def print_ekaterinburg():
      print('Средняя зарплата в Екатеринбурге:', average_salary, 'рублей.')

  print_moscow()
  print_ekaterinburg()
  ```
  выводит:
  ```python
  Средняя зарплата в Москве 150000 рублей.
  Средняя зарплата в Екатеринбурге 150000 рублей.
  ```

#### Почему лучше избегать использования глобальных переменных

Большинство программистов согласны, что следует ограничить использование **глобальных переменных** либо не использовать их вовсе.

Причины следующие:

- Глобальные переменные затрудняют отладку программы.
  Значение глобальной переменной может быть изменено **любой инструкцией в программном файле**. Если обнаружится, что в глобальной переменной хранится неверное значение, то придётся отыскать абсолютно все инструкции, которые к ней обращаются, чтобы определить, откуда же идёт плохое значение. В программах с тысячами строк кода сделать это вовсе непросто.

- Функции, использующие глобальные переменные, обычно зависят от этих переменных.
  Если возникнет необходимость применить такую функцию в другой программе, скорее всего придётся перепроектировать эту функцию, чтобы она не опиралась на глобальную переменную.

- Глобальные переменные затрудняют понимание программы.
  Глобальная переменная может быть модифицирована любой инструкцией в программе. И следовательно при необходимости изучения работы какой-то части программы, использующей глобальную переменную, придётся узнавать обо всех других частях программы, обращающихся к этой глобальной переменной.

В большинстве случаев следует **создавать переменные локально и передавать их в качестве аргументов в функции, которым нужно к ним обратиться**.

#### Глобальные константы

Хоть и следует избегать использования глобальных переменных, в программе допускается применение **глобальных констант**. 

**Глобальная константа** - это **глобальное имя**, ссылающееся на **неизменяемое значение**.

Поскольку значение глобальной константы неизменно и постоянно, то следовательно, оно не может изменяться во время исполнения программы и можно не беспокоиться о потенциальных опасностях, связанных с использованием глобальных переменных.

Несмотря на то, что язык **Python** не позволяет создавать настоящие глобальные константы, их можно имитировать при помощи **глобальных переменных**.

Если глобальная переменная не объявляется с использованием ключевого слова `global` внутри функции, то внутри этой функции вы никак не сможете изменить значение этой глобальной переменной.


## Конспект по теме: "Функции с возвратом значения"

### Основные понятия

### Функция с возвратом значения

**Функция с возвратом значения** похожа на функцию без возврата значения тем, что:
- это набор инструкций, выполняющий определённую задачу;
- когда нужно выполнить функцию, её вызывают.

Однако, есть одно отличие, когда функция с возвратом значения завершается, она возвращает значение в ту часть программы, **которая её вызвала**.

Возвращаемое из функции значение используется **как любое другое значение**: оно может быть присвоено переменной, выведено на экран, использовано в математическом выражении (если относится к числовым типам данных) и другие манипуляции.

Ранее я уже сталкивался со многими функциями, которые возвращают значения:
- Функция `int()` - преобразует строку к целому числу и возвращает его;
- Функция `float()` - преобразует строку к вещественному числу и возвращает его;
- Функция `range()` - возвращает последовательность целых чисел `0, 1, 2, ...`;
- Функция `abs()` - возвращает абсолютное значение числа (модуль числа);
- Функция `len()` - возвращает длину строки или списка.

Функцию с возвратом значения пишут точно так же, как и без, но она должна иметь **инструкцию `return`**.

- Общий формат определения функции с возвратом значения в **Python**:
  ```python
  def название_функции():
      тело функции
      return выражение
  ```

Т.е. синтаксис такой функции отличается только наличием `return`.

Значение выражения, которое следует за ключевым словом `return`, будет отправлено в ту часть программы, которая вызвала функцию.

Это может быть переменная либо выражение, например, математическое.

- Напишем функцию, которая удваивает полученное число:
  ```python
  def double_num(num):
      result = num * 2
      return result
  ```

Задача этой функции максимально проста - умножить передаваемое значение переменной `num` на 2 и вернуть другое значение - `result`, результат умножения.

Как работает данная функция?

Первая инструкция в теле функции присваивает значение `num * 2` переменной `result`. 

Затем исполняется инструкция `return`, которая приводит к завершению выполнения функции и отправляет значение из переменной `result`, назад **в ту часть программы, которая вызвала эту функцию**.

- Общий вид программы:
  ```python
  def double_num(num):
      result = num * 2
      return result

  num = int(input('Введите число:'))
  doubled = double_num(num)
  print(doubled)
  ```

Основная программа получает от пользователя одно число и вызывает функцию, передавая значение переменной `num` в качестве **аргумента**.

Значение, которое возвращается из функции `double_num`, присваивается переменной `doubled`.

Но использование инструкции `return` можно оптимизировать, для этого взглянем на функцию `double_num` ещё разок.

- Вот как она выглядит:
  ```python
  def double_num(num):
      result = num * 2
      return result
  ```

Обратим внимание, что внутри этой функции происходит 2 действия: переменной `result` сначала присваивается значение `num * 2` и после этого начинается второе действие - возвращение переменной `result` из функции.

Эта функция хорошо справляется с поставленной перед ней задачей, но её можно упростить.

- Поскольку инструкция `return` возвращает значение выражения, переменную `result` убираем и переписываем функцию вот так:
  ```python
  def double_num(num):
      return num * 2
  ```
Эта версия функции не сохраняет значение `num * 2` в отдельной переменной, а сразу возвращает значение выражения с помощью инструкции `return`. Т.е. делает она ровно то же самое, что предыдущая версия функции, но используя один шаг вместо двух.

Также стоит обратить внимание, что если мы **явно не указываем в функции, какое значение нужно вернуть, то по умолчанию будет возвращено значение `None`**.

- Пример:
  ```python
  def tripled_num(num):
      tripled = num * 3

  print(tripled_num(3))
  ```
  выводит:
  ```python
  None
  ```
  
#### Использование нескольких инструкций `return`

В одной функции может быть **сколько угодно инструкций `return`**.

- Рассмотрим функцию `convert_grade()`, которая переводит стобалльную оценку в пятибалльную:
  ```python
  def convert_grade(grade):
      if grade >= 90:
          return 5
      elif grade >= 80:
          return 4
      elif grade >= 70:
          return 3
      elif grade >= 60:
          return 2
      else:
          return 1
  # основная программа
  grade = int(input('Введите оценку по 100-балльной системе:'))
  print(convert_grade(grade))
  ```

Как можно заметить в функции `convert_grade()` используется 5 инструкций `return`.

Каждая из них возвращает соответствующее значение и завершает работу функции.

Если в функции будет **несколько инструкций `return`**, то сработает только **самая первая из них, после чего функция завершит работу**.

- Пример:
  ```python
  def get_grade():
      return 10
      return 20
  print(get_grade())
  ```
  выводит:
  ```python
  10
  ```
