# Пользовательские функции - Конспект

## Конспект по теме: "Функции без параметров"

### Основные понятия:

Ранее я использовал встроенные в **Python** функции `print()`, `input()`, `int()`, `str()`, `len()` и множество других, но пришло время начать писать свои собственные функции.

Начнём с того, что же такое функция в языке **Python**.

**Функция** - `def` в **Python** - это фрагмент кода, который выполняет определённые операции и возвращает результат этих операций. Она является отдельной, функционально независимой частью программы, выполняющей определённую задачу.

Вся суть в том, что его можно написать один раз и переиспользовать во всей программе.

Приведу простой пример..

Допустим необходимо решить задачу в которой требуется изобразить звёздный прямоугольник размерами 5 строк на 7 столбцов.

- Так бы выглядел самый очевидный вариант кода без использования функций:
  ```python
  print('*******')
  print('*******')
  print('*******')
  print('*******')
  print('*******')
  ```
- Далее, если вспомнить операторы умножения строк на числа, можно было бы придти к такому коду:
  ```python
  print('*' * 7)
  print('*' * 7)
  print('*' * 7)
  print('*' * 7)
  print('*' * 7)
  ```
- И наконец, если вспомнить и про циклы, то код принимает вид:
  ```python
  for _ in range(5):
      print('*' * 7)
  ```

А теперь представим, просто представим, что нам требуется изобразить сразу несколько таких фигур, например 4 штуки.

- Тогда нам пришлось бы повторять данный цикл 4 раза:
  ```python
  for _ in range(5):
      print('*' * 7)

  print()

  for _ in range(5):
      print('*' * 7)

  print()

  for _ in range(5):
      print('*' * 7)

  print()

  for _ in range(5):
      print('*' * 7)
  ```

  Результатом выполнения такого кода будет:
  ```python
  *******
  *******
  *******
  *******
  *******
  
  *******
  *******
  *******
  *******
  *******
  
  *******
  *******
  *******
  *******
  *******

  *******
  *******
  *******
  *******
  *******
  ```
Вроде как данный код вполне себе решает поставленную задачу, но у него есть несколько недостатков. 

Во-первых, данный код может показаться довольно громоздким из-за нескольких повторений одной и той же части кода, отвечающей за вывод прямоугольника.

Во-вторых, если понадобится изменить размеры прямоугольника, придётся менять их аж 4 раза, в каждой части кода, отвечающей за вывод прямоугольника.

Вот здесь нам и пригодятся функции и вместо повторения кода для вывода прямоугольника, можно перенести его в **отдельную функцию** и вызвать её 4 раза.

- Для создания функции напишем такой код:
  ```python
  def draw_box():
      for _ in range(5):
          print('*' * 7)
  ```

- Когда функция создана, чтобы увидеть результат её работы, надо вызвать её по имени:
  ```python
  draw_box()
  ```

- Теперь, чтобы изобразить 4 прямоугольника, можно написать код:
  ```python
  draw_box()
  print()
  draw_box()
  print()
  draw_box()
  print()
  draw_box()
  ```

Благодаря использованию функции код стал короче, читабельнее, а главное, если потребуются иные размеры прямоугольника, достаточно будет изменить только саму функцию `draw_box()`.



#### Именование функций 

Имена функциям назначаются точно также, как переменным. Имя функции должно быть достаточно описательным, чтобы любой человек, который читает или будет читать ваш код, мог догадаться, что именно делает та или иная функция.

**Python** и тут требует соблюдения тех же правил, что при именовании переменных:
1. В имени функции используются **только латинские буквы** a-z, A-Z, цифры и символ нижнего подчеркивания (_);
2. Имя функции **не может** начинаться с цифры;
3. Имя функции **по возможности должно отражать её назначение**;
4. Символы **верхнего** и **нижнего** регистра **различаются**.

Следует помнить, что **Python** это регистрочувствительный язык. Для именования переменных и функций принято использовать стиль **lower_case_with_underscores** (слова из маленьких букв с подчеркиваниями).

Поскольку функции **выполняют действия**, большая часть программистов предпочитает в их именовании использовать **глаголы**.

Например:
- функцию, которая рисует прямоугольник, можно назвать `draw_box()`;
- функцию, которая печатает чек, можно назвать `print_check()`;
- функцию, которая вычисляет заработную плату до удержаний, можно назвать `calculate_gross_pay()`.

Каждое из этих имён даёт описание того, что делает функция.#### Именование функций

#### Синтаксис и объявление функций

В **Python**, как и в других языках программирования, есть особые правила для создания функций. Если их не соблюдать, то интерпретатор не сможет правильно обработать код и, скорее всего, выдаст ошибку.

Для объявления функции используют **ключевое слово `def`**(от англ. define - определить, обозначить).

- В общем виде объявление функции выглядит следующим образом:
  ```python
  def имя_функции (аргументы):
      тело_функции
      return результат
  ```
Здесь:
- `имя_функции` - название, с помощью которого можно вызывать функцию в коде;
- `аргументы` - значения, которые функция принимает на вход. Это поле может быть пустым;
- `тело_функции` - набор инструкций, которые выполняются при вызове;
- `результат` - значения, которые функция возвращает при завершении работы.


- Рассмотрим на примере функцию, которая находит сумму двух чисел и возвращает результат вычислений:
  ```python
  def sum(a, b):
      return a + b
  ```

Также при объявлении функции очень важно убедиться в том, что каждая строка тела функции начинается с **одинакового количества пробелов**, иначе произойдёт ошибка.

- Приведённый ниже код:
  ```python
    def print_greeting():
        print('Доброе утро!')
  print('Сегодня мы будем изучать функции.')
        print('Это мегаважная тема')
  ```
  приводит к возникновению ошибки:
  ```python
  IndentationError: unexpected indent
  ```

  

#### Вызов функции

Как вызвать функцию?

Чтобы вызвать функцию, надо ввести её название и передать аргументы в скобках.

- В общем виде синтаксис вызова выглядит вот так:
  ```python
  имя_функции(аргументы)
  ```

Теперь вызовем функцию для нахождения суммы, которая была создана ранее.

- Вот как это будет выглядеть:
  ```python
  c = sum(a, b)
  ```

Результат значения функции будет записан в переменную `c`.

Стоит помнить, что объявление функции должно **предшествовать** её вызову, т.е. при вызове функции необходимо убедиться, что эта функция была объявлена до вызова.

- Приведённый ниже код:
  ```python
  # вызов
  print_message()

  # объявление
  def print_message():
      print('Я - Андрей,')
      print('санстрайк. ')
  ```
  приведёт к возникновению ошибки:
  ```python
  NameError: name 'print_message' is not defined
  ```
  

## Конспект по теме: "Функции с параметрами

### Основные понятия:

- Вспомним функцию `draw_box()`, которая выводит звёздный прямоугольник с размерами 5 на 7:
  ```python
  def draw_box():
      for i in range(5):
          print('*' * 7)
  ```

Было бы гораздо удобнее, если бы эта функция выводила прямоугольник с произвольными размерами. 

И функции как раз таки могут принимать входные параметры, что делает их более гибкими.

- **Функции с параметрами** объявляются **так же** как и функции без параметров, только с указанием в скобках:
  ```python
  def название_функции(параметры):
      блок кода
  ```

Теперь, зная это, можно переписать предыдущую версию функции `draw_box()` так, чтобы она принимала параметры, задающие высоту и ширину прямоугольника:

- Вот как это будет выглядеть:
  ```python
  def draw_box(height, width): # функция принимает 2 параметра
      for i in range(height):
          print('*' * width)
  ```

Теперь функция `draw_box()` принимает два целочисленных параметра `height` - высота прямоугольника и `width` - ширина прямоугольника, и для её вызова нам нужно обязательно их указать.

- И теперь, чтобы вывести звездный прямоугольник размерами 10 на 15 пишем код:
  ```python
  draw_box(10, 15)
  ```
  выводит:
  ```python
  ***************
  ***************
  ***************
  ***************
  ***************
  ***************
  ***************
  ***************
  ***************
  ***************
  ```

Теперь с помощью данной функции можем в одной программе выводить прямоугольники разных размеров.

- Приведённый ниже код:
  ```python
  draw_box(3, 3)
  print()
  draw_box(5, 5)
  print()
  draw_box(4, 10)
  ```
  выводит:
  ```python
  ***
  ***
  ***

  *****
  *****
  *****
  *****
  *****

  **********
  **********
  **********
  **********
  ```

На место параметров можно подставлять не только целочисленные константы, но и значения переменных:

- Приведённый ниже код:
  ```python
  n = 3
  m = 9
  draw_box(n, m)
  ```
  выводит:
  ```python
  *********
  *********
  *********
  ```

Приведу ещё один пример:

- Функцию `print_hello(n)`, которая принимает одно натуральное число `n` и печатает слово **hello** ровно `n` раз:
  ```python
  def print_hello(n):
      print('Hello' * n)
  ```

- Приведённый ниже код:
  ```python
  print_hello(3)
  print_hello(5)
  times = 2
  print_hello(times)
  ```
  выводит:
  ```python
  HelloHelloHello
  HelloHelloHelloHelloHello
  HelloHello
  ```

Эту функцию можно сделать ещё более гибкой, если передавать в неё ещё один параметр - текст для вывода.

- Вот как это будет выглядеть:
  ```python
  def print_text(text, n):
      print(text * n)
  ```

- Приведённый ниже код:
  ```python
  print_text('Hello', 5)
  print_text('A', 10)
  ```
  выводит:
  ```python
  HelloHelloHelloHelloHello
  AAAAAAAAAA
  ```

Параметры функций часто называют **параметрическими переменными**.

### Разница между аргументами и параметрами

**Аргумент** - это **любая порция данных**, которая передаётся в функцию, когда функция **вызывается**. 

**Параметр** - это **переменая**, которая **получает аргумент**, переданный в функцию.

- Для функции `draw_box(height, width)`:
  ```python
  def draw_box(height, width):
      for i in range(height):
          print('*' * width)
  ```

Параметрами являются переменные `height` и `width`.

- И в момент вызова данной функции:
  ```python
  height = 10
  draw_box(height, 9)
  ```
  аргументами являются `height` и `9`.

Иногда вместо параметров и аргументов говорят о **формальных параметрах** и **фактических параметрах**. 

**Формальные параметры** - это переменные, которые пишутся при описании функции.

**Фактические параметры** - это то, что реально подставляется при вызове функции.

### Внесение изменений в параметры

Когда **аргумент** передаётся в функцию, параметрическая переменная функции будет ссылаться на значение этого аргумента. Однако любые изменения, которые вносятся в параметрическую переменную, не будут влиять на аргумент.

- Приведённый ниже код:
  ```python
  def draw_box(height, width):
      height = 2
      width = 10
      for i in range(height):
          print('*' * width)
  n = 5
  m = 7
  draw_box(n, m)
  print(n, m)
  ```
  выводит:
  ```python
  **********
  **********
  5 7
  ```

В теле функции вносятся изменения в значения параметрических переменных `height` и `width`, однако это никак не повлияло на значение переменных `n` и `m` из основной программы, которые передавались в качестве аргументов в функцию `draw_box()`.

Т.е. если в теле функции уже заранее заданы значения параметрических переменных, то даже если передать новые значения в функцию, функция всё-равно перезапишет те значения, что были внутри неё и использует именно их, а новые потеряются.
